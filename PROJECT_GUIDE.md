# 网站导航系统 - 项目文件指南

本指南旨在帮助您了解项目的结构、每个文件的作用以及如何对其进行修改。

## 1. 项目概览与结构

```text
navsite/
├── public/              # 前端静态资源（HTML, CSS, JS, 图片）
├── server/              # 后端核心代码（Node.js/Express）
├── database/            # 数据库文件目录
├── .env                 # 环境变量配置文件
├── package.json         # 项目依赖与脚本配置
└── START.md             # 项目启动说明
```

---

## 2. 前端文件说明 (public/)

这些文件主要负责用户界面（UI）和客户端逻辑。

| 文件名 | 作用 | 目标用户 | 修改教程 |
| :--- | :--- | :--- | :--- |
| **index.html** | 首页 | 普通访问者 | 修改首页布局、版头和页脚。 |
| **admin.html** | 管理后台 | 管理员 | 修改后台管理界面布局、报表样式等。 |
| **login.html** | 登录页面 | 管理员 | 修改登录框外观。*(现已移除硬编码凭据)* |
| **submit.html** | 网站提交页 | 想要录入网站的用户 | 修改提交表单的字段。 |
| **category.html** | 分类详情页 | 访问者 | 修改按分类显示网站的样式。 |
| **search.html** | 搜索结果页 | 访问者 | 修改搜索结果的展示方式。 |
| **about.html** | 关于页面 | 访问者 | 修改关于本站的信息。 |
| **css/main.css** | 全局样式 | 开发者 | 修改全站的主色调、字体、间距等。 |
| **js/main.js** | 通用前端逻辑 | 开发者 | 修改时钟、搜索等全局功能。 |

## 3. 后端文件说明 (server/)

这些文件负责处理 API 请求、业务逻辑和数据库交互。

| 文件名/目录 | 作用 | 目标用户 | 修改教程 |
| :--- | :--- | :--- | :--- |
| **app.js** | 程序入口 | 开发者 | 配置中间件、启动端口、路由挂载。 |
| **database.js** | 数据库连接配置 | 开发者 | 修改 SQLite 数据库连接参数。 |
| **routes/** | API 路由层 | 开发者 | 定义新的接口路径（例如 `/api/new-feature`）。 |
| **services/** | 业务逻辑层 | 开发者 | 编写核心业务代码（例如计算统计数据）。 |
| **dao/** | 数据库交互层 (SQL) | 开发者 | 编写或修改 SQL 查询语句。 |
| **middleware/** | 中间件层 | 开发者 | 修改权限验证（Auth）或数据验证逻辑。 |

---

## 4. 数据库说明 (database/)

*   **navsite.db**: 这是一个 SQLite 数据库文件，存储了所有的网站数据、分类信息、管理员账号和审核记录。
*   **如何查看/修改**：可以使用 `DB Browser for SQLite` 等工具打开此文件查看原始数据。

---

## 5. 配置文件

*   **package.json**: 定义了项目的名称、版本、启动脚本（如 `npm start`）和所需安装的库。
*   **.env**: 存储敏感配置信息（如服务器端口、JWT 密钥）。**请勿将其直接上传到公开仓库。**

---

## 6. 如何修改项目？

### 想要修改 UI？
1. 在 `public/` 目录下找到对应的 `.html` 文件。
2. 在 `public/css/main.css` 调整样式。
3. 刷新页面即可看到更改。

### 想要增加一个新接口？
1. 在 `server/routes/` 下创建一个新的路由文件或在现有文件中添加。
2. 在 `server/services/` 编写对应的处理逻辑。
3. 在 `server/app.js` 中注册该路由。

### 想要修改数据库结构？
1. 修改 `server/database.js` 或相关初始化代码来执行 `ALTER TABLE` 语句。
2. 更新对应的 DAO 文件以支持新的字段。
